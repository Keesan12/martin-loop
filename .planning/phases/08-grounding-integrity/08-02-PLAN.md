Wire `scanPatchForGroundingViolations` into the `runMartin` loop at the VERIFY phase so that:
1. After each attempt completes (PATCH → VERIFY transition), if a patch diff is available, scan it against the grounding index
2. If violations are found, append a `grounding.violations_found` ledger event
3. The scan result's `resolvedFiles` and violation count are persisted to the ledger payload

Current state: `scanPatchForGroundingViolations` exists in grounding.ts and is imported in index.ts but is never called from `runMartin`. The `grounding.violations_found` LedgerEventKind exists but is never used in the main loop.

This plan adds the grounding scan call after the VERIFY phase evaluates verification results, and adds a runtime test to confirm the ledger event fires.

```typescript
function buildPatchDiff(result: MartinAdapterResult, changedFiles: string[]): string | undefined {
  // Use structured diff stats to build a minimal diff header if no raw diff is available
  if (result.execution?.changedFiles?.length) {
    // Build a synthetic diff header from changed file list
    return result.execution.changedFiles
      .map((file) => `--- a/${file}\n+++ b/${file}\n@@ -0,0 +1 @@\n+`)
      .join("\n");
  }
  if (changedFiles.length > 0) {
    return changedFiles
      .map((file) => `--- a/${file}\n+++ b/${file}\n@@ -0,0 +1 @@\n+`)
      .join("\n");
  }
  return undefined;
}
```
