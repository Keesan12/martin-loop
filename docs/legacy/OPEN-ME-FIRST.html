This repo includes both the hosted SaaS control-plane code and a local dashboard you can open with no install. If you just want to see something immediately, start with the local dashboard.

Before launching any run, set guardrails first in martin.config.yaml (or pass --config), then use CLI flags only for one-off overrides.

What’s inside

  • Set guardrails first: defaults come from martin.config.yaml (or --config), then CLI flags override for a single run.
  • Budget caps: set budget.maxUsd, budget.softLimitUsd, budget.maxIterations, and budget.maxTokens in the config file.
  • Governance routing: set governance.destructiveActionPolicy, governance.telemetryDestination, and governance.verifierRules in the config file.
  • Verification overrides: pass --verify only when you need a one-off verification plan for a single run.
  • Hosted API demo auth: the control plane now expects x-martin-api-key; deployment should set MARTIN_CONTROL_PLANE_ADMIN_API_KEY and MARTIN_CONTROL_PLANE_WORKSPACE_API_KEY.
  • The Martin runtime and benchmark harness in `packages/*` and `benchmarks/`
  • The hosted SaaS control plane in `apps/control-plane/`
  • A no-install local dashboard in `apps/local-dashboard/`
  • A stakeholder-ready visual pack in `docs/demo/mockups/`
  • Testing and deployment guides in `docs/` and `deploy/`
