The Ralph Loop Problem — Honest Technical Assessment

What the Ralph Loop problem actually is

  • Context bloat: Conversation history accumulates until ~150K tokens. Quality degrades after ~80K. Agent "forgets" the original objective.
  • Oscillation: Agent alternates between two wrong solutions. Attempt 1 breaks test A. Attempt 2 fixes A but breaks B. Attempt 3 reverts A. Infinite cycle.
  • Shallow failure feedback: Agent receives "tests failed" but not which line, which assertion, or what the actual diff is vs. expected.
  • No convergence signal: Without measuring distance-to-correct, the system can't tell if it's getting closer or farther away.
  • Token waste per attempt: If each attempt regenerates 5K tokens of context that doesn't change between attempts, you're burning money on repetition.

What Martin Loop currently solves vs. does not

  • ✅ Context bloat: Session isolation (fresh subprocess). distillContext() caps prior attempts at 3. No conversation history accumulation.
  • ✅ Budget runaway: Hard iteration + USD limits with real enforcement.
  • ✅ Model escalation: change_model intervention routes to capable model on repeated failure.
  • ❌ Oscillation: detectRepeatedFailure() only catches consecutive same-class failures. A↔B cycle runs to budget exhaustion.
  • ❌ Shallow feedback: Raw stderr text. No file:line:assertion extraction. Agent must re-read files to find the error itself.
  • ❌ Convergence measurement: No semantic similarity check between attempts. No "is this getting better?" signal.

The 3 Missing Pieces (Ranked by Impact)

1. Structured Error Extraction

When verification fails, parse stderr for file:line:message. Pass that to the next prompt as a structured block instead of raw text. The agent then knows exactly where to look, not just that something failed.

Impact: Reduces attempts by 30–50% on TypeScript/test failures. Agent doesn't waste tokens re-reading all files to find the error.

2. Oscillation Detection

Hash each attempt's summary (or use a simple fuzzy diff). If attempt N is semantically similar to attempt N-2, the agent is cycling. Trigger change_model immediately. Window: check last 4 attempts for A→B→A→B pattern.

Impact: Eliminates the most common infinite loop pattern. Exit early instead of burning remaining iterations.

3. Progress File (PROGRESS.md)

After each attempt, write a structured PROGRESS.md to the working directory. Include: what was tried, what failed specifically, current hypothesis. The agent reads this at the start of each attempt — structured persistent memory without context bloat.

Impact: The agent builds a real hypothesis chain instead of re-deriving the problem from scratch each attempt.
