# Claim Freeze Matrix

## Purpose
Prevent launch claims from outrunning runtime truth.

| Claim area | Allowed now | Freeze / soften until proven | Proof needed |
|---|---|---|---|
| Basic loop controls | Yes | — | golden path demos |
| Hard budget enforcement | Soften | Until budget v3 exact/estimated modes land | admission + settlement tests |
| Hallucination solved | No | Replace with “grounding checks” | grounding challenge set |
| Exact savings | Only where exact accounting exists | Everywhere else | saved-spend methodology + exact mode |
| Finance-grade dashboard | Soften | Until read model is fully truthful | traceable metrics |
| Safe enterprise controller | No | Until leash v2 certified | safety challenge set |
| Smart failure routing | Soften | Until evidence-backed routing is stable | routing tests + challenge set |

## Rule
If a claim cannot be tied to a specific passing test and artifact, it must be softened or removed.
