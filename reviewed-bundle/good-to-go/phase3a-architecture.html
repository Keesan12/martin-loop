Architecture

Five practical layers around the agent loop.

MartinLoop does not replace the coding agent. It wraps the loop with admission, adapter execution, safety checks, verification, and durable local evidence.

1. Task Contract
Objective, verifier plan, repo root, allowed paths, denied paths, acceptance criteria, workspace, project, and budget.
2. Policy And Budget
Defaults load from martin.config.yaml. CLI flags override config values. Budget preflight can reject attempts before execution.
3. Agent Adapters
Claude CLI, Codex CLI, direct-provider, and stub adapters normalize execution results into the core runtime contract.
4. Safety And Verification
Verifier commands, file scope, approval-boundary changes, secret-like values, grounding, and patch decisions determine whether work is kept.
5. Persistence
The CLI writes JSONL records under ~/.martin/runs/. Repo-backed runs can also persist contracts, ledgers, diffs, and rollback artifacts.
