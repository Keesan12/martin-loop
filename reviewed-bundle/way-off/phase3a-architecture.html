Architecture Explainer
What MartinLoop actually does under the hood.
Five governance layers stack between your prompt and the outside world. Every AI coding agent runs inside this envelope — whether you're on the free OSS tier or enterprise. The layers are what make the guarantee real.
How It Works in 3 Steps
1
You define the rules
Write a martin.policy.yaml — budget, providers, allowed file paths. Check it into your repo. Governance is now code.
→
2
MartinLoop enforces them
Every agent run passes through 5 governance layers: admission control, safety leash, budget governor, verifier, and signed audit trail. All before anything touches your codebase.
→
3
You get proof
A cryptographically signed receipt: cost, grade, diff, test results. On failure: filesystem rollback with zero files written. Either way, you know exactly what happened.
The Governance Stack — 5 Layers
Each layer runs in sequence. Every AI agent run must pass through all five before anything changes in your repository.
ALL TIERS
Layer 1
Admission Control + Policy Check
Budget preflight runs before any model is called. If the estimated cost exceeds your configured limit, the run is rejected immediately — zero tokens spent. Policy-as-code enforced from martin.policy.yaml.
ALL TIERS
Layer 2
Safety Leash — 14 Trap Classes
Pre-execution blocking across 14 adversarial categories: scope drift, filesystem violation, credential access, prompt injection, hallucinated completion, recursive loop injection, and more. 63/63 gate checks passing. Zero false positives.
ALL TIERS
Layer 3
Budget Governor — Hard USD Stop
Real-time spend tracking against your configured cap. When the limit is hit, the subprocess is killed. Not warned — killed. Delta re-prompting on retry reduces token usage by sending only what changed, not full context.
ALL TIERS
Layer 4
Verifier — Grade from Real Tests
Runs your configured test command after each patch. Derives A/B/C grade from actual test output — not the model's self-assessment. Failed verifier = filesystem rollback with zero files written. 8/10 verified completion on the public benchmark.
ALL TIERS
Layer 5
Signed Audit Trail — Ed25519
Every accepted attempt produces a tamper-evident proof bundle: Ed25519 signature, cost provenance, patch diff, grounding scan, test results. Replayable offline. Verify with martin audit verify.
PRO+ ONLY
Enterprise Layer
Control Plane — Dashboard, SIEM, SSO
Pro tier unlocks the live cost dashboard, SIEM-ready event streaming (OCSF/CEF), OpenTelemetry spans, and compliance export reports. Enterprise adds SSO/SAML, custom policy engine, SLA, and dedicated security review.
