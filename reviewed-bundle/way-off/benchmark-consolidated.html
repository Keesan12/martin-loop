Why MartinLoop Costs Less
30–60% cost reduction. Here's the mechanism.
Two verified optimizations combine to cut token usage and model cost significantly — both backed by shipped code (SLICE-04, SLICE-08).
🔄
Delta Re-Prompting
Retry attempts send only what changed — not the full conversation history. This cuts token usage on attempt 2+ significantly compared to full-context re-prompts used by Ralph/Ralphy.
SLICE-04 · Shipped
🤖
Cheap-Model Classifier
A lightweight classifier routes trivial tasks to cheaper models (Claude Haiku-tier) before touching expensive models. Simple changes don't need Sonnet/Opus.

Methodology — How to Reproduce

All benchmark numbers are derived from benchmarks/comparative/history/latest.md in the public repository. The test suite is bun run benchmark from a clean clone — no special configuration required.

Each runner (MartinLoop, Ralphy, Ralph) is given the same 10 task contracts from the same starting state. Cost is measured from actual API billing counters. Correct outcomes are validated against declared acceptance criteria. Verified patches require passing the configured verifier command.

Reproduce this report: git clone github.com/Keesan12/MartinLoop && bun install && bun run benchmark

What we don't claim: We do not claim "100% success rate." The honest number is 10/10 correct outcomes with 8/10 verified completion — and we show both. The 2 patches that didn't pass the verifier bar are correctly flagged as incomplete, not shipped.
